## Problem Statement

Segmented publish treats archive markdown, index pages, watchlist pages, and optional weekly digest as one publish boundary. Most paths are snapshotted before mutation, but the watchlist branch writes pages first and only then records their bytes in `snapshots`.

If a later publish step fails, rollback can restore watchlist pages to the already-mutated bytes instead of the pre-run bytes. In the same area, `publisher/watchlist_pages.py` still uses plain `Path.write_text` for public markdown pages while other publisher surfaces use the shared atomic writer.

## Goal

Make watchlist publish side effects transaction-safe inside the existing publish boundary:

- snapshot all watchlist destinations before mutation
- write watchlist markdown through the shared atomic-write primitive
- prove rollback restores or removes watchlist pages when a later publish step fails

## Existing Coverage / Deduplication

- `_rollback_paths()` already restores bytes or deletes newly created paths from `snapshots`.
- `investo._internal._io.write_atomic` already owns the temp-sibling + replace contract. Reuse it; do not add another atomic helper.
- Existing publish rollback tests cover archive/publish failures but not the watchlist-after-write snapshot gap.

## Scope Boundary

In scope:
- Predict affected watchlist paths before `update_watchlist_pages()` and `write_daily_impact_page()` mutate files.
- Snapshot per-term watchlist pages, watchlist index, and `daily.md` before write.
- Replace production watchlist page `write_text` calls with `write_atomic`.
- Add rollback tests for existing watchlist pages and newly created watchlist pages.

Out of scope:
- No watchlist matching/grouping/public-copy change.
- No weekly digest policy change.
- No new transaction-manager abstraction.
- No fsync/durability expansion beyond the existing shared helper.
- No historical backfill.

## Stage Decision

Functional Design: skip. This is a bounded publish-boundary defect repair over existing publisher/orchestrator contracts.

NFR Requirements: skip. Atomicity is already an existing publisher quality requirement; this unit closes an implementation gap without a new dependency, source, secret, network call, or runtime cost.

## Fixed Contracts

### Snapshot Contract

Before any watchlist page is written, the publish stage must snapshot every destination the watchlist publish branch can mutate.

Rollback rules:

- Existing path before publish failure: restore exact pre-run bytes.
- Missing path before publish failure: remove the newly created path.
- Parent pruning remains owned by `_rollback_paths()`.

### Atomic Write Contract

`src/investo/publisher/watchlist_pages.py` must use the shared atomic writer for production public markdown writes:

- per-term accumulation pages
- watchlist index page
- daily impact page

### Path Planning Contract

Snapshot planning must not rely on post-write return values as the only source of truth. Add a pure helper that predicts affected watchlist paths from the current matches and the same slug/grouping rules used by the writer.

The plan must cover every watchlist public markdown destination before any mutation:

- per-term accumulation pages derived from the matches
- the watchlist index path, even when the index body is generated by a private helper
- the daily impact page path written by `write_daily_impact_page()`
