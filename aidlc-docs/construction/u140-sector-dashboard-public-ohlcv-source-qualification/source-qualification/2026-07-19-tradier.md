# u140 Source Fact Sheet — Tradier Market Data API

**Checked**: 2026-07-19
**Candidate**: Tradier Brokerage API historical market data
**Disposition**: Reject under current written terms for public Pages
**Probe status**: Not run; the candidate failed the public-release rights gate first

## Source Facts

| Field | Value |
| --- | --- |
| Source owner | Tradier Brokerage Inc.; its API Agreement identifies DTN as the market-data feed used for API quotes |
| Data family | Historical daily/weekly/monthly prices for US equities and ETFs; separate real-time and delayed market-data surfaces also exist |
| Official docs | `https://docs.tradier.com/reference/brokerage-api-markets-get-history` |
| Endpoint | `GET https://api.tradier.com/v1/markets/history?symbol={symbol}&interval=daily&start={date}&end={date}`; one symbol per request |
| Auth | Required Bearer token tied to a Tradier Brokerage account. Sandbox access also requires a brokerage account and paper-trading token. |
| Cost and no-paid evidence | Tradier says it does not charge brokerage account holders for API access and lists a USD 0 Lite brokerage plan. This is personal account access, not a standalone free public-product license. Business/public integrations must contact Tradier and have no published no-cost entitlement. |
| Rate and request budget | Market Data limit is 120 requests/minute in production and 60 requests/minute in sandbox, per token. Twelve symbol-specific history requests fit the preliminary technical budget. |
| Format | JSON when requested with `Accept: application/json`; the historical response contains a `history.day` array |
| Key fields | `date`, `open`, `high`, `low`, `close`, and `volume` |
| Required symbols | Tradier documents standard symbols for US equities and ETFs, making `XLC, XLY, XLP, XLE, XLF, XLV, XLI, XLB, XLRE, XLK, XLU, SPY` plausible; exact availability was not probed after rejection |
| Update cadence | Brokerage equities market data is real-time and sandbox data is delayed 15 minutes, but the reviewed public docs do not pin the daily-history completion time needed for the u140 freshness gate |
| Historical range | Tradier says reasonable date ranges usually cover the security's entire lifetime, sufficient in principle for 63 daily bars |
| Adjustment semantics | The historical guide warns that data may not be dividend adjusted and provides no separate adjusted-close field. Split/corporate-action semantics are not fully specified. |
| Attribution | When data or information is displayed, Tradier asks for contextual attribution, typically a linked Tradier logo or “Powered by Tradier.” Attribution does not replace Partner approval for a public-release application. |
| Caching and raw retention | The reviewed individual API docs and API Agreement do not establish an Investo public caching, raw-retention, or derived-field grant. No payload or fixture is stored. |
| License/public display | Tradier's official FAQ says APIs are entitled for personal use unless the user is a Tradier Partner. Its authentication guide directs public-release applications to become Partners, while its business page sends integrations to sales for pricing. Investo has no Partner approval or no-cost public-display agreement. |
| Existing Investo overlap | No Tradier runtime, test, config, dependency, credential, or workflow surface exists. Existing Yahoo history/u138 remains operational evidence only and does not supply Tradier rights. |
| Proposed `source_name` | None; rejected before design |
| Proposed adapter path | None; rejected before design |
| Routing surfaces | None; no registry, spec, tier, market-window, segment, config, diagnostics, fixture, token, or workflow change |
| Degradation behavior | Not applicable; no runtime path is registered |

## Dated Primary-Source Evidence

Checked on 2026-07-19:

1. `https://docs.tradier.com/reference/brokerage-api-markets-get-history`
   - documents the Bearer-authenticated single-security `/v1/markets/history`
     endpoint, daily/weekly/monthly intervals, and start/end dates;
   - says reasonable ranges usually cover the security's lifetime.
2. `https://docs.tradier.com/docs/historical-data`
   - identifies historical daily candles as OHLCV;
   - warns that historical data may not be dividend adjusted.
3. `https://docs.tradier.com/docs/historical`
   - documents the JSON `history.day` response and its date, open, high, low,
     close, and volume fields.
4. `https://docs.tradier.com/docs/rate-limiting`
   - sets `/markets` limits at 120 requests/minute in production and 60 in
     sandbox, enforced per access token.
5. `https://docs.tradier.com/docs/faq`
   - says API access is not charged to Tradier Brokerage account holders;
   - limits non-Partner API entitlement to personal use and directs application
     distributors to become Tradier Partners;
   - confirms coverage of US equities and ETFs and 15-minute sandbox delay.
6. `https://docs.tradier.com/docs/authentication`
   - requires a Partner relationship for a public-release application and reserves
     Partner credentials, OAuth application registration, and refresh tokens for
     approved Partners.
7. `https://docs.tradier.com/docs/attribution-guidelines`
   - asks displayed data to carry contextual Tradier attribution, usually a linked
     logo or “Powered by Tradier”; it does not state that attribution authorizes
     public release.
8. `https://production.tradier.com/individuals/web`
   - lists a USD 0 Lite brokerage plan with API access.
9. `https://production.tradier.com/businesses/fintechs`
   - describes account-holder API access as personal use and directs businesses to
     sales for integration pricing.
10. `https://api.tradier.com/v2/applications/agreements?key=api_agreement`
    - identifies Tradier Brokerage Inc.'s API Agreement and DTN-supplied quote
      market data, without granting individual users public publication rights.

Tradier is technically capable of serving the sector universe, but its free API
entitlement belongs to individual brokerage account holders. Investo Pages is a
public-release application, which Tradier routes through Partner approval and
business pricing. Attribution guidance cannot be treated as that approval.

## Deduplication Evidence

A repository scan across `src/`, `tests/`, `scripts/`, `.github/`, `CONTRIBUTING.md`,
dependency configuration, AIDLC plans, and requirements found no Tradier adapter,
registration, source spec, tier/window/segment route, fixture, credential,
dependency, or workflow. Adding it would introduce a new brokerage-account Bearer
token trust boundary while leaving the public-use gate failed.

## Gate Decision

| Binding gate | Result | Reason |
| --- | --- | --- |
| Official terms and public derived-display rights | Fail | Non-Partner API access is personal use; public-release applications require Partner approval and no Investo agreement exists |
| Free access for 63+ daily OHLCV bars | Preliminary pass | Account holders receive uncharged API access, history usually spans the security lifetime, and the response exposes OHLCV |
| Exact 12-symbol coverage and freshness | Not run | Fail-fast after the binding rights failure; daily-history freshness and adjustment semantics are also not fully pinned |
| Five bounded GitHub Actions probes | Not run | A brokerage account, token, and provider-payload testing are unjustified after rejection |
| No paid plan/trial/secret exposure | Fail | Individual access requires a brokerage account and secret token; public Partner pricing has no documented no-cost entitlement |
| Public/raw retention contract | Fail | Public caching, raw retention, and derived-field rights are not granted in the reviewed individual contract surfaces |

**Disposition: `reject under current written terms`.** Reconsider only with a Tradier
Partner agreement that explicitly covers Investo's public derived fields,
attribution, caching, raw retention, GitHub Pages delivery, and zero API cost. No
account/token, local probe, GitHub Actions probe, adapter, fixture, Pages output, or
TECH-DEBT item is created.
