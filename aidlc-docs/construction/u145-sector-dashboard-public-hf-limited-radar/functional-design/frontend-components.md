# Public Radar Components: `u145 sector-dashboard-public-hf-limited-radar`

**Date**: 2026-07-22
**Status**: Complete
**Surface**: static MkDocs Pages Markdown; no client-side data fetch

## 1. Surface Contract

The page answers “which supported sector samples are leading or lagging relative to SPY?”
without implying consolidated U.S. market coverage. It is deterministic Markdown generated
from `PublicSectorDashboardSnapshot`. No JavaScript calls HF Data Library, no API key reaches
the browser, and no raw OHLCV history is embedded.

## 2. Component Order

| Order | Component | Visibility |
| ---: | --- | --- |
| 1 | `LimitedCoverageBanner` | always |
| 2 | `FreshnessAndCoverageHeader` | always |
| 3 | `RadarSummary` | partial/normal with at least 8 ranked sectors |
| 4 | `SectorRadarTable` | always; unavailable rows retained |
| 5 | `TextRegimeQuadrant` | partial/normal |
| 6 | `MissingCoverageCallout` | always under HF v1 |
| 7 | `MethodAndAttribution` | always |

## 3. Component Definitions

### C1. `LimitedCoverageBanner`

First viewport, fixed visible text:

- `제한 공개 베타`
- `IEX venue sample 기준`
- `10/11 섹터 사용 가능 · XLRE unavailable`
- `미국 전체시장 거래량 또는 자금 흐름이 아님`

The banner is never collapsed, generated by an LLM, or omitted for a fresh snapshot.

### C2. `FreshnessAndCoverageHeader`

Fields:

- market `as_of` date with `미국 정규장 마감 기준`;
- freshness badge: `최신 / 지연 / 확인 불가`;
- coverage: available and comparable counts out of eleven;
- benchmark: `SPY (IEX venue sample)`;
- source: `HF Data Library`;
- policy id: `sector-regime-v1`.

It does not show provider retrieval timestamps, key status, account information, or raw errors.

### C3. `RadarSummary`

Deterministic content only:

- top two and bottom two comparable sectors by relative rank;
- regime counts among comparable sectors;
- explicit rank denominator;
- one fixed qualification: relative strength in this IEX sample is observational, not an
  investment recommendation or whole-market sector breadth.

No causal narrative, flow claim, volume ranking, or predicted return appears.

### C4. `SectorRadarTable`

Columns:

1. rank (`—` if unavailable)
2. sector/ticker
3. availability
4. regime
5. IEX price return 1D
6. IEX excess vs SPY 5D
7. IEX excess vs SPY 21D
8. 5D relative acceleration
9. IEX realized volatility 20D
10. IEX max drawdown 20D

XLRE always appears with `provider 미지원`, `—` metrics, and no color that implies measured
weakness. Transient missing sectors use a distinct `일시 수집 불가` label.

### C5. `TextRegimeQuadrant`

Four text groups: `주도 / 둔화 / 회복 / 부진`. Only complete comparable records appear.
Unavailable sectors are listed below as `분류 불가`; they are never placed in `부진`.

The primary 10 bps band is shown. Sensitivity results remain machine/test evidence unless a
later UX unit explicitly approves them.

### C6. `MissingCoverageCallout`

Fixed explanation:

- HF v1 does not catalog XLRE;
- no substitute ETF or inferred series is used;
- rankings compare only the currently comparable sector samples;
- coverage expansion requires a separately qualified source/version.

### C7. `MethodAndAttribution`

Contains:

- concise formulas/horizons and the non-overlapping acceleration definition;
- IEX venue scope and the fact that IEX volume is excluded from score/rank;
- HF Data Library CC BY 4.0 attribution/link;
- mandatory IEX historical-data attribution/link;
- informational-use disclaimer and methodology version.

Both attribution entries are structural required fields, not optional prose.

## 4. State Behavior

### Fresh partial

Render all components. This is the expected HF v1 state.

### Warming up

Render banner/header/table/callout/method. Show only metrics whose history contract is met;
hide summary/quadrant and label the remaining warm-up requirement.

### Insufficient before first publish

Do not create or replace the public page. The workflow remains red with redacted diagnostics.

### Held last-good

Keep the last validated metric bytes and original `as_of`. The visible delivery wrapper adds
`데이터 갱신 지연` and a stable reason category, never a provider error string. It does not
relabel stale metrics as current.

## 5. Formatting and Accessibility

- Semantic headings and Markdown table headers; no color-only state encoding.
- Korean state text plus ticker identity in every row.
- Mobile-first table may horizontally scroll through existing MkDocs behavior; the banner,
  freshness, sector, rank, and regime remain first.
- Percent values use deterministic half-even formatting; missing values use `—` plus a text
  reason in the availability column.
- Links have descriptive labels rather than bare URLs.

## 6. Forbidden Surface Content

- API key, auth status detail, provider error body, response/request URL, raw OHLCV rows
- `전체 거래량`, `시장 전체`, `자금 유입/유출`, `수급`, unqualified `거래강도`
- volume-based score, dollar volume, actual flow, constituent breadth, earnings actual
- XLRE proxy, inferred rank, or placement in a measured regime
- buy/sell/overweight/underweight advice, target, forecast, or LLM-generated causal narrative
- Telegram controls or links implying Telegram delivery exists
