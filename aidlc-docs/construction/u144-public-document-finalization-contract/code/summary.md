## Delivered boundary

Default segmented publishing now has one typed transition:

```text
generated Briefing + immutable E1 context
  -> assemble every text/asset producer
  -> one public projection
  -> bounded region containment
  -> read-only trust validation
  -> SHA-256 sealed E5 documents + terminal notifier DTOs
  -> staged E6 artifact promotion and exact-byte publication
```

is called once by production. No archive/index/quality/notifier consumer can
mutate or rediscover public Markdown after the seal.

## Fixed contracts

- Presentation defects are contained by owned region and cannot remove a
  segment. Numeric/entity/compliance/disclaimer/structure/notification-summary
  trust failures remain typed `trust_blocked` outcomes.
- A known generation absence or trust block may commit valid siblings, but is
  `content_completeness=partial`, dispatches Pages, then exits 2. Zero
  survivors exit 1. Complete-content notifier-only failure remains exit 0.
- Optional supplement omission keeps the marker shell, contributes no E5
  artifact ID, and therefore cannot enter the E6 promotion manifest.
- The notifier consumes only `PublicNotificationSummary` values derived from
  terminal layout bytes and E1 coverage. Generated `Briefing.market_summary`
  is unreachable after finalization.
- Git-preparation I/O failures restore snapshotted public bytes and remove the
  run staging root. Commit/push failure retains the existing explicit
  `PublisherGitError` recovery boundary.
- Publisher shares neutral algorithms through `models`/`_internal`; u114 AST
  guards prohibit publisher-to-briefing/orchestrator sibling imports.
