### GAP-001: FR-001 AC-2 / US-001 AC-2 — only 1 of 6 source categories has an adapter

**Requirement**: FR-001 lists 6 source categories — 주가/지수, 크립토 시세, 거시 지표(FRED 등), 연준 캘린더, 주요 기업 뉴스, 실적 캘린더 — and US-001 AC says "카테고리별 소스 1개 이상".
**Status**: ⚠️ Partial — `fomc-rss` covers "연준 캘린더" only. 5 categories remain unimplemented.
**Impact**: Medium. The plugin contract is complete (any future adapter is a single-file addition), but **the MVP cannot ship a useful daily briefing with only the FOMC feed** — there's no price data, no crypto, no macro, no general news, no earnings.
**Why this is not a blocker for u1 stage close**: u1's design contract was "framework + 1 reference adapter", not "all categories". The Plan §0 explicitly scopes Step 8 to `fomc_rss.py` as the PoC. Adding the remaining adapters is per-source PR work that rides on the contract u1 just locked in.

**Proposed Action**: Add the 5 missing adapters as discrete tasks. Each is self-contained per the 4-step procedure. Suggested ordering by signal-density per minute of dev work:

| Suggested adapter | Category | Source | Free-tier confirmed |
|-------------------|----------|--------|--------------------|
| `coingecko.py` | crypto price | `https://api.coingecko.com/api/v3/...` | Yes (no key, ≤30 req/min free) |
| `yfinance_quote.py` | price/index | yfinance Python lib | Yes (unofficial; data via Yahoo public endpoints) |
| `fred_series.py` | macro | `https://api.stlouisfed.org/fred/...` | Yes (free key) — note: registers a key, must declare in PR per US-009 |
| `yfinance_earnings.py` | earnings | yfinance | Yes (same as quote) |
| `news_rss_aggregator.py` | news | aggregate Reuters/AP/SEC EDGAR RSS feeds | Yes (RSS, no auth) |

These are not part of u1's stage; they are MVP-completing tasks for the development plan. Recommended priority: **High** for the briefing to be useful, but **not blocking** the u1 cross-check sign-off.
