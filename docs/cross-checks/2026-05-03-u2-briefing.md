## Gaps Analysis

### GAP-001: NFR-007 AC-7.7 test-traceability drift

**Requirement**: `nfr-requirements.md` AC-7.7 states no `eval`, `pickle.loads`, or `exec` on response data inside u2, pinned by `tests/unit/briefing/test_no_eval.py`.

**Status**: ⚠️ Partial. The implementation is clean by inspection and manual grep, but the named automated test file does not exist. `code/summary.md` also records AC-7.7 as passive verification, which conflicts with the NFR document.

**Impact**: Low. No runtime bug was found, but future regressions are not pinned exactly as the NFR document says.

**Proposed Action**: Add a small `tests/unit/briefing/test_no_eval.py` grep/AST test, or update `nfr-requirements.md` to mark AC-7.7 as passive inspection. Adding the test is preferable because it aligns with the original AC.

### GAP-002: NFR-002 AC-2.5 wording is stricter than implemented test scope

**Requirement**: `nfr-requirements.md` AC-2.5 says `"CLAUDE_CODE_OAUTH_TOKEN"` is not a literal anywhere in `src/investo/briefing/`.

**Status**: ⚠️ Partial / documentation drift. `claude_code.py` intentionally mentions the env var in docstrings explaining that Python does not read it. The executable-code test strips docstrings/comments and verifies the runtime code does not reference the token.

**Impact**: Low. Runtime secret handling is correct; the drift is between NFR prose and the implemented, more precise executable-source check.

**Proposed Action**: Update AC-2.5 wording to "not present in executable code" and keep docstring mentions allowed in negative/explanatory context. No production code change needed.
